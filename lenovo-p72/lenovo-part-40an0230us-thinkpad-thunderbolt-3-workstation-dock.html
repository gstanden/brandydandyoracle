Lenovo Part #40AN0230US ThinkPad Thunderbolt 3 Workstation Dock

Background

This page is about setting up the Lenovo Part # 40AN0230US ThinkPad Thunderbolt 3 Workstation Dock ("the Dock") on a Lenovo P72 ThinkPad Mobile Workstation running both Windows 10 and Ubuntu 18.10 each installed on a separate NVME in the two NVME bays with which the P72 is equipped.

Summary

My testing found that this dock will NOT work with Ubuntu Linux 18.10. Therefore, I do not recommend it for use on the Lenovo P72 Mobile workstation. If you still are interested in trying to use this then my notes are below. Otherwise I recommend using a different dock, especially if you are primarily using a dock to enable multiple external displays with Ubuntu Linux in a dual NVME configuration running Windows 10 and Ubuntu Linux.

Install the Dock

BEFORE configuring Ubuntu 18.10 on a second NVME it is best to completely install and configure the Dock with only the OEM Windows 10 NVME installed. The Dock should best be fully configured and installed in Windows 10 before adding a second NVME to the system and configuring Ubuntu. The main reason for this is that after connecting the Dock fully, Windows 10 will automatically detect and prompt to update the firmware for the Dock and associated Thunderbolt systems, and if Ubuntu 18.10 is already installed in a dual-NVME configuration then issues may be encountered during firmware auto-restart which cause Ubuntu to be booted instead of Windows.

Also note that if Windows 10 does not automatically prompt to updated the the Dock firmware then you should launch the Lenovo Vantage application (should be already installed out-of-the-box, bundled with the OEM P72) and run the "Check Updates" to get all updates including firmware for the Dock. Note that the Dock must be powered up and connected to the ThinkPad P72 Mobile Workstation before running the "Check Updates" with Lenovo Vantage app.

If you already have both Windows 10 and Ubuntu 18.10 installed in a dual NVME setup, then prior to beginning firmware update of the Dock, ensure that boot is set in the BIOS to boot the Windows 10 NVME first so that on reboot the Windows 10 will be automatically selected.

During the firmware update of the Dock, add-on external additional monitors may blank out for several seconds. Also, if during the firmware update you get any problems with display looking disastrous (jumbled unusable fragmented video output) just WAIT awhile the firmware update should complete eventually. Do NOT reboot or shutdown manually or anything else until you have got at least a normal login and desktop back on at least the ThinkPad P72 Mobile Workstation built-in display (external monitors may not be working if the firmware issue had problems). If during the firmware update you get the fragmented jumbled screen on the P72 main built-in display, which eventually resolves to a normal screen (but with external monitors not working) then launch the Lenovo Vantage app and click on "Check for Updates" and if the firware update for the Dock is listed, run the firmware update again.

Just a personal note here, during my firmware update of the Dock, a transformer exploded in our neighborhood exactly when Windows 10 was rebooting for the firmware update reboot step. Note that a transformer has exploded in our neighborhood only twice that I can recall in the 23 years that we have lived here, and this was the second time. Fortunately, I have everything on UPS so when the power flickered momentarily the UPS took over and of course the laptop has a battery too. Nevertheless it was unsettling and the timing of the transformer explosion was to put it mildly, incredible, that it coincided exactly with a firmware update!

The other personal note is that when I did my firmware update of the Dock, I already had Ubuntu 18.10 installed and configured on the second NVME, and d'Oh! I had forgotten to set boot to Windows 10 first in the BIOS (I use Ubuntu 18.10 most of the time) so on the firmware reboot it went into Ubuntu 18.10. I hit the kill switch and shut off the laptop and then did a boot into Windows 10 and everything turned out OK in the end, although I did get the jumbled graphics on the first boot into Windows 10 after the firmware update and had to wait a good 10 minutes or so. I was VERY tempted to hit the kill button for 10 seconds and shut the laptop down and boot but that's usually a really bad idea on a firmware update (since system may reboot more than once) so I waited for what seemed like an eternity for 10 minutes, and also noted that the HD activity light was active, and was active in an erratic way that didn't look like just background activity, so I waited, and waited, and waited, and when I was most tempted to hit the kill switch, I instead went and poured a cup of tea from the pot downstairs, and put it in the microwave for 20 seconds, then came back upstairs, and voila! at least the built-in monitor was normal and usable and displaying the login screen.

So I logged in and went to Lenovo Vantage and found that Vantage was still saying firmware for the Dock needed update. So I am guessing that some or all of the firmware update might have been rolled back. So I ran the firmware update for the Dock from Lenovo Vantage and this time made sure Windows 10 booted first on reboot (set this in BIOS and TEST it on a couple reboots to be sure that it is ALWAYS Windows 10 that boots by default) and this time when it was all done, Lenovo Vantage showed everything up to date, no updated pending, and the Dock firmware at latest rev.
