KVM Oracle Clone

Contents

  1. 1 Create the KVM Guest Clone
  2. 2 Review Cloned Guest XML
  3. 3 Edit OpenvSwitch XML Stanzas
  4. 4 Start Cloned KVM Guest
  5. 5 Edit Guest 70-persistent-net.rules File

This page document the steps necessary to take a fully-built KVM guest running Oracle standalone separation environment, which here means linux user "grid" owns the Oracle ASM instance and the Oracle listener, and linux user "oracle" owns the Oracle RDBMS. This is cloning a fully built system which has a database and an ASM instance already running. The storage for the ASM instance and the RDBMS is provided by an Linux SCST iSCSI SAN from yet another KVM guest, oracle651.

Create the KVM Guest Clone

The initial clone was created as shown below.

gstanden@vmem1:~$ man virt-clone
gstanden@vmem1:~$ virt-clone --connect qemu:///system --original oracle631 --name oracle632 --file /var/lib/libvirt/images/oracle632-0.img --file /var/lib/libvirt/images/oracle632-1.u00.img
Allocating 'oracle632-0.img' | 30 GB 01:40
Allocating 'oracle632-1.u00.img' | 20 GB 01:30

Clone 'oracle632' created successfully.
gstanden@vmem1:~$
